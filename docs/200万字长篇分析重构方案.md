# 200 万字长篇小说分析 — 重构方案

## 核心原则

**不要压缩文本，要压缩认知。** 中间产物不是摘要文本，而是结构化事件数据库。因果链不丢。

## 新架构：四阶段

```
200万字原文
    │
    ▼
═══════════ 阶段 1：流式索引（程序化，廉价） ═══════════
  ├─ 按章节/段落边界预切分为 ~400 个 Chunk（每 Chunk ~5000字）
  ├─ 对每个 Chunk 做轻量实体提取（已有的 program-indexer.ts）
  ├─ 构建实体字典：角色/地点/组织/道具/概念
  └─ 产出：chunks.json + entity_index.json
    │
    ▼
═══════════ 阶段 2：重叠窗口事件提取（AI，核心步骤） ═══════
  ├─ 5个窗口 × ~45万字（重叠5万字缝合）
  ├─ 每个窗口：AI 提取结构化事件列表
  │   {id, 章节, 角色[], 类型, 导致→[事件ID], 被导致←[事件ID]}
  ├─ 在重叠区匹配相同事件 → 合并去重
  └─ 产出：event_graph.json（5000-10000条事件，含因果边）
    │
    ▼
═══════════ 阶段 3：价值采样 + 道/气分析（AI，并行） ═══════
  ├─ 从事件图谱中自动识别：高潮节点、转折节点、伏笔-回收对
  ├─ 按情绪曲线做价值采样 → 选取 30-50 个核心切片用于深度分析
  ├─ 并行运行：体验流标注 + 消融测试 + 势能追踪
  └─ 产出：experience_curve.json + ablation_results.json + tension_analysis.json
    │
    ▼
═══════════ 阶段 4：DNA V2 蒸馏（程序化） ═══════
  ├─ 输入：事件图谱 + 实体字典 + 体验曲线 + 消融结果 + 势能分析
  ├─ buildNovelDNA()  → 六层 DNA（道/气/骨/肉/引擎/风险）
  └─ 产出：novel_dna_v2.json（< 3万字）
```

## 各阶段详解

### 阶段 1：流式索引

**目标**：不调 AI。用程序化手段把 200 万字变成可检索的结构。

**做的事**：
1. `fs.createReadStream` 流式读取，按章节标题正则（"第X章"）或固定 5000 字切分
2. 复用已有的 `program-indexer.ts`（中文 n-gram 实体提取）对每个 Chunk 提取实体
3. 构建全局实体字典：合并同名实体、检测别名（"韩立"/"韩道友"/"厉飞雨"）
4. 产出约 400 个 Chunk + 一份实体字典（几千条实体）

**API 调用**：0（全部程序化）

---

### 阶段 2：重叠窗口事件提取

**目标**：这是替代当前 Step 0-3 的核心步骤。不再产出"文风报告""情节报告"等文本摘要，而是产出结构化事件。

**5 个窗口**：

| 窗口 | 覆盖范围 | 大小 |
|------|---------|------|
| W1 | Chunk 001-090 | ~45万字 |
| W2 | Chunk 070-160 | ~45万字（重叠 W1 的 070-090） |
| W3 | Chunk 140-230 | ~45万字 |
| W4 | Chunk 210-300 | ~45万字 |
| W5 | Chunk 280-400 | ~45万字 |

**每个窗口内做的事**：
1. 将该窗口的所有 Chunk 原文 + 实体字典装入上下文
2. AI 提取结构化事件：

```json
{
  "id": "E0142",
  "chapter": 32,
  "type": "拜师",
  "participants": ["韩立", "墨居仁"],
  "location": "七玄门",
  "description": "墨居仁正式收韩立为徒，传授长春功",
  "causes": ["E0101"],      // 导致此事件的前置事件
  "effects": ["E0170","E0260"],  // 此事件导致的后置事件
  "tension_change": 2,      // 紧张度变化 -5~+5
  "emotion": "希望",
  "foreshadowing_of": null, // 如果是伏笔，指向回收事件
  "foreshadowing_from": "E0095" // 如果是回收，指向埋设事件
}
```

3. 5 个窗口**并行**执行（`Promise.all`）
4. 在重叠区（如 W1 的 070-090 与 W2 的 070-090），用**实体+章节号+类型**三重匹配识别同一事件，合并去重

**为什么重叠 5 万字？** 防止一个事件被切成两半。重叠区的同一个事件在两个窗口都会被提取，合并时取信息更完整的版本。

**API 调用**：5 次（5 个窗口并行）

---

### 阶段 3：价值采样 + 道/气分析

**目标**：从事件图谱中选出最值得深度分析的切片，运行已有的道/气模块。

**采样策略**（不平均，按价值选）：

| 类别 | 选取条件 | 数量 |
|------|---------|------|
| 高潮点 | tension_change ≥ 4 或因果边密集 | 10-15 个 |
| 转折点 | 事件 type 含"背叛/揭露/死亡/觉醒" | 5-8 个 |
| 伏笔-回收对 | foreshadowing_of 非空 | 5-10 对 |
| 呼吸段落 | tension_change ≈ 0 且在两个高潮之间 | 5-8 个 |
| 用户锚点 | 用户标注的"最打动我的片段" | 3-5 个 |

总计约 **30-50 个核心切片**，而非均匀的 200 个。

**并行运行**（复用现有模块）：
- 体验流标注 → 仅对采样的 30-50 个切片
- 消融测试 → 同上
- 势能追踪 → 基于事件图谱（不需原文）

**API 调用**：9-16 次（比原来减少 80%，因为只分析精选切片）

---

### 阶段 4：DNA V2 蒸馏

**目标**：从事件图谱 + 实体字典 + 体验曲线 + 消融结果中程序化构建六层 DNA。

**输入不再是文本摘要，而是结构化数据**：

| DNA 层 | 数据来源 |
|--------|---------|
| 道（主情绪场） | 事件图谱的情绪分布 + 体验曲线高点 + 用户确认 |
| 气（维持方法） | 事件图谱中 tension_change≈0 的事件模式 |
| 骨（必须结构） | 因果链中入度+出度最高的事件节点 |
| 肉（增强体验） | 消融测试 muscle 分类 |
| 填充（节奏） | 消融测试 filler_a/b + 事件密度分布 |
| 引擎（势能） | 事件图谱中 tension_change 的周期模式 |
| 风险 | 长程因果链（跨>100章）的比例 |

**产出大小**：< 3 万字（因为所有输入都是结构化数据，不是文本报告）

---

## 与原方案对比

| 维度 | 原方案 | 新方案 |
|------|--------|--------|
| 中间产物 | 5 份 markdown 报告（累计 2-8 万字） | 1 个事件图谱 JSON（结构化） |
| 因果链 | 丢失（报告是自然语言，无法追踪因果） | 保留（事件节点有 causes/effects 边） |
| 跨卷伏笔 | 无法识别（报告只说"存在伏笔"） | 可追踪（foreshadowing_of/foreshadowing_from 字段） |
| 阶段 A API 调用 | 22-70 次 | 5 次（窗口事件提取） |
| 阶段 B API 调用 | 9-16 次 | 9-16 次（但只分析精选切片） |
| DNA 质量 | "报告总结的总结" | "事件图谱的结构分析" |
| 200万字总耗时（估算） | 31-86 次 × 10s = 5-14 分钟 | 5 次 × 30s + 10 次 × 10s = 4 分钟 |

## 实施步骤

### Step 1：实现事件提取 prompt（新文件）
- `src/lib/ai/prompts/event-extraction.ts`
- AI 输入：Chunk 原文 + 实体字典 → 输出：结构化事件列表 JSON
- 关键要求：必须标注 causes（前置事件ID）和 effects（后置事件ID）

### Step 2：实现重叠窗口编排器（新文件）
- `src/lib/source-processing/windowed-event-extractor.ts`
- 5 窗口并行 → 重叠区事件合并
- 事件 ID 去重策略：实体+章节+类型匹配

### Step 3：实现价值采样器（新文件）
- `src/lib/source-processing/value-sampler.ts`
- 从事件图谱中按 tension_change / 类型 / 伏笔关系选取 30-50 个核心切片

### Step 4：更新 dao-pipeline.ts
- 替换输入：事件图谱替代文风/情节/角色报告
- `buildNovelDNA()` 改为从结构化事件数据中推断

### Step 5：更新 context-assembler.ts
- 生成时注入事件图谱中的因果链上下文（而非文风报告全文）

### 保留不改的
- 体验流标注器（experience-annotator.ts）
- 消融测试器（ablation-tester.ts）
- 势能追踪器（tension-tracker.ts）
- 5 层生成管线
- 6 维审查系统
