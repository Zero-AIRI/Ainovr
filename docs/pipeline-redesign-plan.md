# 统一分析管线重构方案

## 背景

当前有三条独立管线，只有基础管线接入 UI。用户提出将三者合并为一条统一管线，采用"双层切片 + 全书记忆图谱"架构。

## 架构概览

```
                     rawText
                        │
        ┌───────────────┴───────────────┐
        │   第一遍：构建全书记忆图谱      │
        │                               │
        │  小切片 → 逐片事件提取(并行)    │
        │        → 事件链接(一次AI)       │
        │        → 全局 EventGraph        │
        └───────────────┬───────────────┘
                        │
        ┌───────────────┴───────────────┐
        │   第二遍：深度分析              │
        │                               │
        │  大切片 → 逐片深度分析(并行)     │
        │        → 汇总+事件功能标注(一次AI)│
        │        → 全局报告               │
        └───────────────┬───────────────┘
                        │
                     DNA 压缩
                (代码填充 + AI 定性)
```

## 7 步详细流程

### Step 1：小切片（程序化，无 AI）

- 输入：rawText
- 按灵活尺寸切分（5-10 章或几千字/片，不固定数值）
- 纯代码分割，不调 AI
- 输出：小切片列表

### Step 2：逐切片事件提取（AI，并行）

- 输入：每个小切片的原文
- AI 提取该切片内的结构化事件和实体
- 实体由 AI 自行识别，不设独立实体识别步骤
- 所有切片并行调用，失败重试 2 次，仍失败则跳过（该切片事件留空）
- 输出：per-slice 事件片段列表（JSON）

### Step 3：事件链接 → 全书记忆图谱（AI，一次调用）

- 输入：所有 per-slice 事件片段
- AI 构建全局事件图谱：因果链、伏笔配对、事件分组
- 一次 AI 调用完成（小切片事件摘要是紧凑 JSON，总量可控）
- 输出：全局 EventGraph（全书记忆图谱）

### Step 4：大切片（程序化，无 AI）

- 输入：rawText
- 按模型上下文窗口切分（基于 settings 里的 maxContextTokens）
- 纯代码分割
- 输出：大切片列表

### Step 5：逐大切片深度分析（AI，并行）

- 输入：每个大切片原文 + 全局 EventGraph
- AI 做机制型逆向工程分析，输出结构化 JSON
- **核心原则**：分析目的是指导生成实践，不是文学评论。分析不出来就留 null，不写套话
- 体验标注（多 persona 读者视角）合并到此步骤
- 输出维度：
  - 文风机制（句式规律、对话密度、描写策略——可复现的操作）
  - 叙事机制（章节结构、节奏控制、视角切换——怎么做）
  - 角色机制（弧线阶段、行为模式、关系变化——可模仿）
  - 情绪机制（情绪走向、积累-释放模式）
  - 读者体验（沉浸度、节奏感——多 persona 视角）
  - 叙事约束（世界观规则、角色限制、情节约束）
- 输出格式：结构化 JSON

### Step 6：汇总 + 事件功能标注（AI，一次调用）

- 输入：所有大切片分析 JSON + 全局 EventGraph
- AI 一次性完成：
  1. 合成全局报告（文风演化轨迹、叙事结构、角色弧线、情绪曲线、约束清单）
  2. 标注每个事件的叙事功能（推进剧情/舒缓节奏/营造氛围/角色塑造/伏笔/过渡）
  3. 识别未完成事件（open）和未回收伏笔（dangling）
  4. 检测文风不一致/演化（不强制统一，如实记录变化）
- 输出：汇总报告 + 事件功能标签

### Step 7：DNA 压缩（代码填充 + AI 定性）

- 纯代码填充：从 EventGraph + 汇总数据计算道/气/结构/引擎的量化部分
- AI 定性：提炼风格引擎、技法库、风险标注
- 一份 DNA 包含全部（基因 + 技法 + 风险）
- 机制型输出，所有结论指向"生成时怎么做"
- 输出：NovelDNA（唯一输出，覆盖旧 novelDna / novelDnaV2 / techniqueSamples）

## 设计决策汇总

| # | 决策 | 结论 |
|---|------|------|
| 1 | 管线架构 | 一条统一管线，双层切片 |
| 2 | 小切片尺寸 | 灵活（5-10 章或几千字），不固定 |
| 3 | 大切片尺寸 | 按模型 maxContextTokens 自适应 |
| 4 | 事件提取方式 | 逐小切片 AI 提取，并行 |
| 5 | 实体识别 | 事件提取 AI 自行识别 |
| 6 | 事件链接 | 一次 AI 全量链接 |
| 7 | 事件功能标注 | 汇总阶段统一标注 |
| 8 | 体验标注 | 合并到大切片深度分析 |
| 9 | 分析风格 | 机制型逆向工程，不写套话，分析不出就留 null |
| 10 | 文风不一致 | 如实记录演化轨迹，不强制统一 |
| 11 | 未完成事件/伏笔 | 显式标注 open/dangling |
| 12 | 水字数判断 | 主观，由事件功能标签体现，不做硬性判定 |
| 13 | 样本选取 | 去掉 |
| 14 | 消融测试 | 改为事件功能标注（推进/舒缓/氛围/角色/伏笔/过渡） |
| 15 | 势能追踪 | 吸收到汇总（EventGraph tensionChange + 大切片 emotionalArc） |
| 16 | DNA 产物 | 一份全包（基因+技法+风险），机制型 |
| 17 | DNA 构建方式 | 纯代码填充结构 + AI 定性归纳 |
| 18 | 并行容错 | 重试 2 次后跳过 |
| 19 | 用户配置 | 零配置，一键到底 |
| 20 | 部署 | 仅本地 |

## 涉及的代码变更

### 新建文件
- `src/lib/source-processing/unified-pipeline.ts` — 统一管线编排器（7 步）
- `src/lib/source-processing/pipeline-types.ts` — 管线类型定义（小切片/大切片 JSON schema）

### 修改文件
- `src/lib/store/source-processing.ts` — 重构为统一管线状态管理
- `src/components/source/SourceNovelDetailView.tsx` — 简化 UI（一键 + 统一进度）
- `src/lib/source-processing/windowed-event-extractor.ts` — 改为逐切片事件提取（去掉窗口机制，去掉 TextChunk 依赖）
- `src/lib/ai/prompts/event-extraction.ts` — prompt 适配逐切片提取

### 废弃/合并
- `dna-compressor.ts` → 合并到 DNA 步骤（Step 7）
- `value-sampler.ts` → 废弃（去掉样本选取）
- `program-indexer.ts` → 废弃（事件提取 AI 自行识别实体）
- `entity-classifier.ts` → 废弃
- `ablation-tester.ts` → 改为事件功能标注（Step 6 内）
- `tension-tracker.ts` → 吸收到汇总
- `experience-annotator.ts` → 合并到大切片分析（Step 5）
- `dao-pipeline.ts` → 各阶段拆分嵌入统一管线
- `basic-pipeline.ts` → 被统一管线替代

### 保留不动
- `smart-slicer.ts` → 大切片步骤可能复用其输出格式
- `style-extractor.ts` 等 → 逻辑吸收到大切片分析 prompt，文件可后续清理
- 5 层生成管线 → 不变（读取 NovelDNA 接口不变）
- 所有 API routes → 可能需要新增 1-2 个，旧的逐步废弃

## 验证方案

1. **10 万字小说**：小切片 ~33 个 → 并行事件提取 33 次 → 1 次链接 → 1 个大切片 → 1 次深度分析 → 1 次汇总 → DNA
2. **200 万字小说**：小切片 ~667 个 → 并行事件提取 667 次 → 1 次链接 → ~3 个大切片 → 3 次并行深度分析 → 1 次汇总 → DNA
3. **无章节短篇**：小切片按字数切，流程不变
4. **容错验证**：模拟 API 失败，验证重试+跳过不阻塞管线
5. **生成管线兼容**：验证新 DNA 格式能正常驱动 5 层生成
